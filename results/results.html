How to read this chart
  • The x-axis is the framework version (one point per cohort), oldest on the left. When more than one model is in view a second tick line shows the model, and the line breaks across a model change — we never draw a cross-model "trend" (only like-model-to-like is comparable).
  • Each point is the median across that cohort's N runs for one arm; the faint vertical bar is the inter-quartile (Q1–Q3) spread. Hover a point for n and the exact band.
  • mandrel = clone carries .agents/ and runs /plan→/deliver; control = bare task prompt, no scaffolding. The gap between the two arms is what the scaffolding buys (value) or charges (cost).
  • Value side (Quality, Autonomy, Maintainability, Security) — higher is better. Cost side (Tokens, USD, Overhead ratio) — lower is cheaper; cheaper only "wins" at equal quality. The caption under the chart states the direction and what good looks like for the selected metric.

What the dimension numbers mean
  • Quality (value · 0–1 · ▲ better) — fraction of the frozen acceptance suite that passes, cross-checked by an LLM judge. 1.0 = fully correct; 0 = no runnable app.
  • Planning (value · 0–1 · ▲ better) — how well the plan predicted reality (story-count, re-plan, file-footprint). — for control: it authors no plan.
  • Autonomy (value · 0–1 · ▲ better) — 1.0 = fully unattended; each human intervention drops it (0.5 = one, 0.33 = two). Any value < 1.0 is a finding.
  • Maintainability (value · 0–1 · ▲ better) — 1.0 = clean, well-structured, easy-to-change code; lower scores indicate complexity or structural debt.
  • Security (value · 0–1 · ▲ better) — 1.0 = no known vulnerabilities or unsafe patterns; lower scores indicate secrets, injection risks, or other findings.
  • Tokens (cost · ▼ cheaper) — total tokens the run spent. Lower is cheaper, but only meaningful at equal Quality.
  • Overhead (cost · ▼ cheaper) — ceremony tokens per shippable-codegen token. Control ≈ 0; the mandrel−control gap is the ceremony tax.
