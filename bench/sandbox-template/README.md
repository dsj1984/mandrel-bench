# Ephemeral benchmark sandbox

This repository is a **throwaway, per-cell workspace** generated by
`mandrel-bench` (see `bench/driver/sandbox.js` in the benchmark harness). It
was materialized from the harness's in-repo `bench/sandbox-template/` baseline
(plus an optional per-scenario seed layer) and pushed here as the cell's
baseline commit.

## What this is

- One private GitHub repository per benchmark **cell** (a `scenario × arm`
  pair), created immediately before the cell's runs and deleted immediately
  after (`create → seed → run(N serial) → destroy`).
- Nothing in this repository is hand-authored or meant to persist. Between
  each of the cell's serial runs, `main` is force-reset back to this baseline
  commit so every run starts from the same clean tree.
- If you are reading this on GitHub outside of an active benchmark run, the
  cell either crashed before teardown or the janitor sweep has not yet run —
  it is safe to delete.

## Repo naming

Every ephemeral sandbox repo is prefixed `bench-sbx-` — this prefix is
**reserved**: nothing else under the benchmark's configured GitHub owner
should use it, since the harness (and its janitor sweep) treats any
`bench-sbx-*` repo under that owner as safe to delete unattended.

## Provenance

- Harness: `mandrel-bench` (`bench/run.js`, `bench/driver/sandbox.js`).
- Template source: `bench/sandbox-template/` in the harness repository.
